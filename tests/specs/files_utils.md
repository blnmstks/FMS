# Spec: `app/utils/files`

Чистые утилиты файловой системы. Без доменных знаний и без вызовов LLM/API.
`read_text_file` / `save_to_vault` / `read_from_vault` — текстовый I/O Obsidian-vault.

## `delete_files(paths: list[str]) -> None`

### Contract
Удаляет файлы по списку путей. Отсутствующий путь игнорируется
(`Path(path).unlink(missing_ok=True)`) — функция не падает на уже удалённых/несуществующих
файлах. Каталоги не трогает. Нужна уборке шага 13: зафейленные клипы и их QC-кадры после того,
как бит прошёл проверку.

### Invariants
1. Каждый существующий файл из `paths` удалён после вызова.
2. Несуществующий путь — no-op (без исключения).
3. Пустой список — no-op.

### Test cases (unit, `tmp_path`)
- **удаляет существующие**: создать 2 файла → `delete_files([...])` → оба исчезли.
- **игнорирует отсутствующие**: `delete_files([tmp/"нет.txt"])` не бросает.
- **смесь**: один существует, один нет → существующий удалён, исключения нет.
